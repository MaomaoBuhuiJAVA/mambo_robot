# Mambo 机器人制作说明

本文说明整机所需**硬件**、**接线方式**、以及**软件/代码**如何配置与烧录。具体引脚以仓库内 `esp32/esp32_mambo.ino` 为准。

---

## 1. 系统组成

| 模块 | 作用 |
|------|------|
| **主控（SBC）** | 香橙派 4 Pro 等：运行 `mambo_robot`（视觉、语音、Web、与 ESP32 通信） |
| **运动与安全 MCU** | **ESP32-S3**：电机驱动、传感器采集、LD2402 雷达串口解析，**115200 串口**与 SBC 交换 JSON 状态 / 文本指令 |
| **USB 声卡×2（典型）** | 麦克风录音、扬声器播放（ALSA `plughw` 编号以 `arecord -l` / `aplay -l` 为准） |
| **摄像头** | USB 或板载，对应 `config.hpp` 中 `kCameraIndex` |

---

## 2. ESP32 端硬件清单（与固件一致）

固件注释：`esp32/esp32_mambo.ino` 头部。

| 器件 | 接口 | 说明 |
|------|------|------|
| **ESP32-S3 开发板** | — | `Serial` USB 调试；`Serial1` 接 SBC；`Serial2` 接雷达 |
| **L298N** | GPIO | 双路直流电机：`ENA,IN1,IN2` 左轮；`ENB,IN3,IN4` 右轮 |
| **MPU6050** | I2C | IMU 采样（本项目已移除 INA226 电压电流采集） |
| **红外跌落 ×3** | GPIO | 左/中/右，代码为 **上拉默认下拉输入**，任一路为高视为**悬崖**（见下文逻辑） |
| **海凌科 LD2402** | UART | **115200**，固件按**文本行**解析 `distance:XX`，非工程模式二进制帧 |

---

## 3. ESP32 引脚接线表

GPIO 定义见 `esp32/esp32_mambo.ino`：

```
I2C:   SDA = GPIO8,   SCL = GPIO9   → MPU6050
与 SBC: Serial1  RX = GPIO16, TX = GPIO17  （交叉接 TTL：SBC_TX→ESP_RX16，SBC_RX←ESP_TX17，共地）
雷达:  Serial2  RX = GPIO18, TX = GPIO19  → LD2402（模块 TX→ESP_RX18，模块 RX←ESP_TX19）
跌落:  CLIFF_L=5, CLIFF_M=6, CLIFF_R=7   → 传感器输出（与模块电平匹配，共地）
电机:  ENA=10, IN1=11, IN2=12, IN3=13, IN4=14, ENB=15  → L298N 逻辑侧（与 ESP32 同 3.3V/5V 按模块要求）
```

### 3.1 与香橙派串口（TTL）

- **波特率**：115200，8N1（与 `utils.hpp` 中 `SerialManager` 一致）。
- **交叉接线**：香橙派 **UART_TX** → ESP32 **GPIO16 (RX)**；香橙派 **UART_RX** ← ESP32 **GPIO17 (TX)**；**GND 共地**。
- **注意**：确认香橙派侧 UART 引脚与使能（部分板子需在设备树/overlay 中打开 `ttySx`）。程序默认端口为 `config.hpp` 的 **`kSerialPort`**（示例为 `/dev/ttyS7`），**以你板子实际串口节点为准**（可用 `dmesg | grep tty` 排查）。

### 3.2 LD2402

- 供电按模块手册；**UART 交叉**接 ESP32 `Serial2`（18/19）。
- 固件期望上行文本类似：`distance:70`（单位 cm，带换行）。若模块模式不同，需改 `ldReadSerial()` 或模块配置。

### 3.3 红外跌落传感器

代码逻辑：

- `isSafe()`：左/中/右 **任一为高** → 判定**不安全（悬崖）**。
- 引脚模式：`INPUT_PULLDOWN`，空闲为低。请选用**输出能与该逻辑匹配的传感器**（例如检测到悬空/无反射时输出高），或在外部加反相/上拉调整。

### 3.4 L298N 与电源

- **逻辑电源**：与 ESP32 IO 电平兼容（多数模块 5V 逻辑可接；以 L298N 模块说明为准）。
- **电机电源**：独立电池/电源，**地与 ESP32/L298N 逻辑地共地**。
- **ENA/ENB**：代码使用 `analogWrite` 调速；默认 `motor_speed=220`。

### 3.5 MPU6050

- **I2C 共线**：SDA/SCL 并接，注意上拉。

## 4. 通信协议摘要（便于自制上位机）

### 4.1 ESP32 → 香橙派（约每 400 ms 一行 JSON + `\n`）

字段由 `Serial1.printf` 发出，`utils.hpp` 解析。示例结构：

```json
{"ax":0.94,"ay":0.30,"az":-0.30,"gx":-1.5,"gy":2.7,"gz":-0.5,"cliff":0,"radar":1,"act":"stop","alert":"","radar_dist":70,"radar_energy":120}
```

- **`alert`**：非空时表示一次性警报（如 `cliff`、`fall`、`dizzy`、`agitated`），主程序消费后清空。
- **`act`**：`forward` / `backward` / `left` / `right` / `stop` / `blocked` / `emergency_back` 等。

### 4.2 香橙派 → ESP32（文本行 + `\n`）

由 `SerialManager::SendCommand` 写入，与 `handleCommand()` 一致：

| 指令 | 含义 |
|------|------|
| `forward` / `backward` / `left` / `right` / `stop` | 运动控制 |
| `speed:0`～`speed:255` | 电机 PWM 上限 |

**超时**：非 `stop` 指令若超过约 **300 ms** 未收到新指令，固件自动停车（`CMD_TIMEOUT`）。

对话逻辑里会在一段时间内周期性重复发送动作指令以“续命”，结束时发 `stop`（见 `dialog_system.hpp`）。

## 5. 香橙派端软件（需要的代码与构建）

### 5.1 仓库目录（与运行相关）

| 路径 | 说明 |
|------|------|
| `src/` | C++ 主程序、`config.hpp` 配置 |
| `models/` | YOLO `.nb`、ONNX 人脸/情绪模型（见 README） |
| `faces/` | 人脸底库图片 |
| `third_party/` | `httplib.h`、`json.hpp`、awnn 等 |
| `eyes-ui/` | 眼睛前端（可选单独 `npm run build`，由 Web 服务挂载） |
| `build.sh` | 编译并可选拉起浏览器全屏 |

### 5.2 依赖（板端环境）

- CMake、OpenCV、libcurl、ALSA、pthread  
- 厂商 NPU 相关库（`CMakeLists.txt` 中的 **NBGlinker、VIPhal** 等）需与香橙派镜像/SDK 一致  

### 5.3 编译运行

```bash
chmod +x build.sh
./build.sh
```

或手动：

```bash
mkdir -p build && cd build && cmake .. && make -j8
export LD_LIBRARY_PATH="$(pwd)/../third_party:$LD_LIBRARY_PATH"
./mambo_robot
```

### 5.4 必改/常改配置（`src/config.hpp`）

| 配置项 | 含义 |
|--------|------|
| `kSerialPort` | ESP32 串口设备节点 |
| `kAlsaRecDevice` / `kAlsaPlayDevice` | 麦克风、扬声器 ALSA 名（`arecord -l` / `aplay -l`） |
| `kCameraIndex` / 分辨率 / 视频流参数 | 摄像头 |
| `kWebPort` | Web 与 MJPEG 端口（默认 8080） |
| API Key 等 | 百度、DeepSeek 等（按你部署填写） |

**安全提示**：密钥勿提交到公开仓库；生产环境用环境变量或独立配置文件更合适。

## 6. ESP32 固件烧录

1. 用 **Arduino IDE** 或 **PlatformIO** 打开 `esp32/esp32_mambo.ino`。  
2. 开发板选择 **ESP32-S3**，端口选择对应 USB 串口。  
3. 上传。上电后 USB `Serial` 应打印 `=== ESP32-S3 Mambo 启动 ===` 及 MPU6050 自检信息。

---

## 7. 音频与麦克风自检（推荐）

仓库 `tools/`：

- `mic_level.sh`：录 2 秒打印与主程序一致的平均绝对值音量。  
- `mic_echo_test.sh`：指定 card 回声环回，确认麦克风与扬声器设备号。

测试前**关闭** `mambo_robot`，避免独占声卡。

---

## 8. 整机检查清单

- [ ] ESP32 USB 串口监视无报错，LD2402 有 `distance:` 类输出（若已接雷达）  
- [ ] 香橙派 `SerialManager` 能打开 `kSerialPort`，控制台有 ESP32 JSON 对应打印  
- [ ] `arecord -l` / `aplay -l` 与 `kAlsaRecDevice`、`kAlsaPlayDevice` 一致  
- [ ] `models/`、`faces/` 路径与 `config.hpp` 一致  
- [ ] 电机空载试转：先低 `speed:`，人离轮系，注意防夹
